Xin review cách giải bài Tính diện tích đa giác lồi bằng công thức Shoelace, nhưng bị TLE ở Test case 14

Chào mọi người, trong Contest diễn ra cuối tuần trước (Weekly Contest #42), ở Bài số 3 yêu cầu tính diện tích đa giác lồi với số đỉnh N ≤ 10^5.

Mình áp dụng công thức dây giày (Shoelace formula) với vòng lặp trên mảng. Complexity lý thuyết là O(N), nhưng nộp thì bị Time Limit Exceeded (TLE) ở Test Case 14, chạy mất 1020ms. Mọi người xem giúp code C++ của mình có tối ưu ở đoạn nào được không ạ? 

#include <iostream>
#include <vector>
#include <cmath>

using namespace std;

struct Point {
    long long x, y;
};

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL); // Mình đã tối ưu I/O rồi

    int n;
    while(cin >> n) {
        vector<Point> pts(n);
        for(int i = 0; i < n; i++) cin >> pts[i].x >> pts[i].y;

        long long area = 0;
        for(int i = 0; i < n; i++) {
            int j = (i + 1) % n; // Phần này có vẻ tốn CPU do phép chia dư?
            area += (pts[i].x * pts[j].y) - (pts[j].x * pts[i].y);
        }
        
        cout << abs(area) / 2.0 << "\n";
    }
    return 0;
}

Đoạn int j = (i + 1) % n; của bạn rất hay bị tốn chi phí rẽ nhánh trên CPU và phép modulo (chia dư) là phép tính chậm nhất.

Vì bạn lặp từ 0 tới n-1, nên j = i+1. Bạn chỉ cần xét trường hợp i = n-1 (lúc đó j = 0) ở bên ngoài là được, tránh phải check modulo n lần.

long long area = 0;
for(int i = 0; i < n - 1; i++) {
    area += (pts[i].x * pts[i+1].y) - (pts[i+1].x * pts[i].y);
}
// Chốt đuôi đa giác
area += (pts[n-1].x * pts[0].y) - (pts[0].x * pts[n-1].y);
